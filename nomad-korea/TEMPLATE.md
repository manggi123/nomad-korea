## 📐 구현 원칙

### SOLID 원칙 적용
- **단일 책임 원칙 (SRP)**: 각 함수는 한 가지만 수행
- **개방-폐쇄 원칙 (OCP)**: 확장 가능하지만 수정 불필요
- **의존성 역전 원칙 (DIP)**: 추상화에 의존

### 함수형 패턴
```typescript
// 파이프라인 패턴
const result = pipe(
  input,
  transformStep1,
  transformStep2,
  transformStep3
);

// 순수 함수
export function pureFunctionExample(input: Input): Output {
  // 부수 효과 없음
  return transformed;
}
```

### Server vs Client 컴포넌트
- **Server Component**: 데이터 페칭, SEO 중요 콘텐츠
- **Client Component**: 상호작용, 상태 관리 (`"use client"`)
